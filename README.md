# HR-платформа для сотрудников, HR-специалистов и администраторов

## О проекте

HR-платформа предназначена для автоматизации взаимодействия между сотрудниками, HR-специалистами и администрацией. Система объединяет в одном интерфейсе работу с задачами, обращениями, обучением, опросами, тестами, форумом, уведомлениями и аналитикой.

Приложение реализовано как полноценная клиент-серверная система и работает с реальной базой данных MySQL через backend API.

## Краткое описание интерфейсного решения

Интерфейс приложения построен в формате единой рабочей среды с боковой навигацией и набором связанных страниц. Для пользователя доступны основные разделы:

- дашборд;
- сотрудники;
- отделы;
- задачи;
- обращения;
- опросы и тесты;
- обучение;
- форум;
- аналитика;
- профиль.

Интерфейс адаптирован под разные роли пользователей. Сотрудник видит только доступную ему информацию, а HR-специалист и администратор получают доступ к управлению сотрудниками, отделами, тестами, задачами, обращениями и общей аналитикой.

Для создания сущностей используются модальные окна. Для отображения метрик применяются карточки, таблицы, графики и прогресс-бары. Для информирования пользователя о событиях реализована система уведомлений.

## Пользовательские сценарии

### Сотрудник

Сотрудник может:

- зарегистрироваться и войти в систему;
- просматривать личный дашборд;
- видеть только свои задачи и отмечать их выполнение;
- проходить назначенные курсы по модулям;
- проходить доступные опросы и тесты;
- создавать обращения, выбирать получателя и вести переписку в формате чата;
- участвовать в общем форуме;
- получать уведомления о новых задачах, тестах, обращениях и других событиях.

### HR-специалист

HR-специалист может:

- просматривать список сотрудников и открывать подробную информацию по каждому;
- добавлять сотрудников и отделы;
- назначать задачи конкретным сотрудникам;
- создавать тесты;
- просматривать прогресс обучения и результаты опросов сотрудников;
- отвечать на обращения и менять их статус;
- работать с общей аналитикой и экспортировать отчёты.

### Администратор

Администратор обладает полным доступом к системе и может выполнять все действия HR-специалиста, а также контролировать структуру платформы и все общие данные.

## Описание реализованной части приложения

На текущем этапе реализованы следующие возможности:

- регистрация и авторизация пользователей;
- разграничение доступа по ролям;
- работа с реальной базой данных вместо mock-данных;
- добавление сотрудников и отделов;
- создание задач для сотрудников и отметка их выполнения;
- создание и обработка обращений с перепиской внутри обращения;
- прохождение опросов и тестов;
- создание тестов для HR и администратора;
- прохождение курсов по модулям с автоматическим завершением;
- просмотр прогресса обучения и результатов опросов других сотрудников для HR и администратора;
- общий рабочий форум с темами и сообщениями;
- система уведомлений;
- аналитические панели и экспорт отчётов в Excel, PDF и Word.

## Архитектура и стек технологий

### Frontend

- React
- Vite
- React Router
- Material UI
- Recharts
- Axios

### Backend

- Node.js
- Express
- MySQL2
- JWT
- Express Validator

### База данных

- MySQL

Приложение построено по клиент-серверной архитектуре:

- frontend отвечает за интерфейс, маршрутизацию, формы, графики и пользовательские сценарии;
- backend реализует REST API, бизнес-логику, проверку прав и работу с базой данных;
- MySQL хранит данные сотрудников, ролей, отделов, задач, обращений, курсов, опросов, уведомлений и другой информации системы.

## Установка и запуск

### Frontend

```bash
cd hr-platform
npm install
npm run dev
```

### Backend

```bash
cd hr-platform-backend
npm install
npm run dev
```

### Production build frontend

```bash
npm run build
```
